#### 3. Analytics

WebLingo uses **PostHog EU Cloud** for product analytics, reliability signals, and privacy-protected session replay on the website and logged-in dashboard.

For the deployed public service, browser analytics requests route through WebLingo's PostHog managed proxy at `metrics.weblingo.app`; while this managed proxy beta is enabled, PostHog routes that proxy traffic through Cloudflare.

PostHog receives pseudonymous event data and dashboard identifiers such as internal user/account IDs, plan/status, and workspace audience. It does not receive payment card details.

Browser capture is configured without autocapture, automatic pageview/pageleave capture, cookies or persistent browser storage, heatmaps, console logs, performance capture, or network request payloads. Session replay is sampled, ignores very short sessions, masks all inputs and page text, disables canvas recording, and drops captured network requests.

Translated pages and translation dictionaries are stored securely on Vercel and Cloudflare infrastructure.
